# Ludens-Flow 后续开发计划 (Roadmap)

## 1. 运行稳定性增强与代码重构
- **核心目标**：精简冗余代码，梳理核心工作流，提升阶段扭转与运行的稳定性。
- **具体方向**：清理如 `orchestrator.py` 等遗留代码，优化图节点 (`graph.py`) 和核心路由 (`router.py`) 的流转逻辑，增加更健壮的异常处理与崩溃恢复机制，确保长流程交互体验。

## 2. 提示词 (Prompt) 优化与结构化输出 (Schemas)
- **特定受众优化**：专门针对 **Unity 开发新人** 进行提示词优化，在对话和代码规划环节给予更易懂、更具实操性的分步引导。
- **阶段工件优化**：深度优化各个阶段核心工件（如 GDD、实现计划等）的大语言模型生成提示词，保障产出质量。
- **结构强约束**：通过引入严格的 Schemas (例如 JSON Schema，或 Structured Outputs 功能)，规范模型输出的工具调用和结构化数据格式，减少脏数据带来的流转中断。

## 3. 独立系统人设与用户人设文件
- **目标**：实现各阶段 Agent 角色与用户设定的高度可定制化与模块化。
- **具体方向**：
  - **Agent 专属人设**：将各个 Agent (如 Design, PM, Eng, Review) 的 System Prompt 和基础“人设”抽离为独立的配置文件，赋予它们更专业独特的“性格”与处理框架。
  - **动态用户画像**：新增“用户人设”文件配置（如用户的编程技术栈、熟练度、表达习惯、重点关注面等），让智能体组能根据不同使用者进行自适应的对话沟通策略转换。

## 4. 模态扩展：图像识别与联网搜索
- **视觉感知能力**：引入多模态大模型 API（如 GPT-4V, Claude 3.5 Sonnet 等），赋予 Agent 接纳和处理视觉输入的能力（比如上传设计草图、游戏实机截图或开发引擎里的报错截图）。
- **外部知识获取**：集成联网搜索工具（Web Search API 插件），打破原生模型的知识库时效性限制，让系统能实时查阅最新的 Unity 文档、解决疑难报错、检索网络上的优秀游戏设计案例进行参考。

## 5. 前端交互界面设计与开发
- **核心目标**：提供更直观的可视化体验，摆脱目前纯 CLI 命令行控制台的交互局限。
- **具体方向**：
  - **工作流可视化**：设计并开发一个图形化前端界面（可考虑使用 Gradio、Streamlit，或更重量级的 React/Vue + FastAPI 架构），在页面上动态展示多智能体的工作转盘/流转状态图。
  - **交互工作台**：实现在前端能进行多轮对话干预，同时用左右分屏等形式支持对核心资产工件 (Artifacts) 进行版本的对比、查看与手动编辑。
