【⚠️ 技术风险】

1. **物理碰撞抖动与高速穿模**  
   在高速移动对象或复杂碰撞体交互场景下，`Rigidbody` 的离散检测（Discrete）可能导致穿透地形或抖动抖动。建议在关键高速对象使用 `Continuous` 或 `Continuous Dynamic` 检测模式，或对 `CharacterController` 进行手动地面检测（Ground Check）逻辑补偿，避免依赖引擎默认斜坡处理。

2. **动画状态机复杂度失控**  
   使用 `Animator` 时，随着状态（State）与过渡条件（Transition）增加，可能产生难以调试的过渡冲突或延迟响应。建议早期规划明确的 State Machine 结构，限制子状态机（Sub-State Machine）嵌套层级；对于简单 2D 动画，可考虑直接使用 `SpriteRenderer` 配合代码驱动帧切换，降低状态机维护成本。

3. **场景加载与内存管理**  
   Game Jam 中常见的场景切换内存泄漏或同步加载导致的卡顿（Frame Hitch）。建议避免在 `Awake`/`Start` 中执行重度初始化（如大量资源加载），改用 `Addressables` 或 `SceneManager` 的异步加载（AsyncOperation）配合 Loading 界面，并注意对象池（Object Pool）中失效对象的及时清理，防止内存冗余。
