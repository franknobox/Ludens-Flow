# Ludens-Flow 前端优先开发计划

> 目标：先定页面逻辑、信息架构、UI/UX 和状态反馈，再接真实后端功能。  
> 范围：只讨论 **ROADMAP 中尚未完成部分的纯前端工作**。

---

## 开发原则

- **产品能力前端先行**：先定入口、页面、状态和反馈，再做后端。
- **基础设施后端先行**：路径审核、schema、runtime 这类底层能力可以先后端验证。
- **先做可验证版本**：页面壳、假数据、状态流先跑通，再接真实接口。
- **先收口再扩展**：不在 UI 还没稳定时同时拉太多后端能力。

## 开发顺序

## P0. 工作台与设置页收口
- **目标**：统一 Workbench / Settings 的页面结构和状态心智。
- **前端重点**：
  - 收口项目栏、设置导航、详情区、滚动规则
  - 补齐 loading / empty / error / success
  - 统一工具页、工作区页、历史项目页的视觉节奏

## P1. 文件写入与权限确认流
- **目标**：把“写文件”做成可见、可确认、可回看的一条用户流程。
- **前端重点**：
  - 工具执行过程卡
  - 文件改动卡
  - 写入前确认弹窗 / 卡片
  - 项目级写入总开关与 workspace writable 的可视化

## P2. 工具能力中心
- **目标**：让工具页从“说明列表”升级为“能力中心”。
- **前端重点**：
  - 工具目录与能力状态
  - 最近执行 / 最近失败 / 最近改动
  - 本轮参考文件 / 修改文件 / 使用工作区

## P3. Unity 工程能力页面化
- **目标**：为 Unity 文件读写、检索、MCP 和编辑器能力提供长期前端承载。
- **前端重点**：
  - Unity 能力入口
  - 工程浏览壳
  - 编辑器 MCP 能力入口壳
  - 高风险能力状态提示

## P4. 结构化结果承接
- **目标**：让前端开始稳定消费结构化结果，而不只显示纯文本。
- **前端重点**：
  - 通用结构化回复视图
  - 不同 Agent 的结构化承接样式
  - 聊天区中的“文本 + 结构卡片”混合展示

## P5. 文案与表达子能力页面
- **目标**：把文案能力做成策划 Agent 的可配置子能力，而不是单独聊天框。
- **前端重点**：
  - 文案任务定义页
  - 风格 preset / 用途 / 字数 / 限制配置
  - 结果页、版本比较、导出入口

## P6. 游戏内模型能力接入页
- **目标**：先定义“游戏内 Agent / Prompt / Tool 配置”在前端的完整路径。
- **前端重点**：
  - 场景模板（NPC、任务、剧情回顾等）
  - 配置表单壳
  - 导出与接入说明区

## P7. 更多引擎兼容页面壳
- **目标**：给 Godot / UE / Blender 等后续引擎兼容预留稳定入口。
- **前端重点**：
  - 引擎兼容总入口
  - 每种引擎的能力卡与状态
  - Godot / UE / Blender 占位详情页

## P8. 协作平台可视化
- **目标**：先定义 GitHub 等协作平台在工作台中的可视化呈现方式。
- **前端重点**：
  - 协作页壳
  - issue / PR / commit / CI 模块
  - 与 Agent 工作流联动入口

## P9. 外部 AIGC 集成页
- **目标**：把外部图片、声音、模型、原型设计能力做成统一入口。
- **前端重点**：
  - AIGC 分类入口
  - 平台能力卡
  - 结果回收占位流程

## P10. Benchmark 与验收可视化
- **目标**：定义 benchmark、核心指标和 release checklist 的前端呈现。
- **前端重点**：
  - 质量/评测页面
  - benchmark 结果卡
  - 指标面板
  - release checklist 页壳

## 推荐推进顺序

### 第一阶段：近期核心产品能力
1. `P0` 工作台与设置页收口  
2. `P1` 文件写入与权限确认流  
3. `P2` 工具能力中心  
4. `P3` Unity 工程能力页面化  
5. `P4` 结构化结果承接

### 第二阶段：中期差异化能力
6. `P5` 文案与表达子能力页面  
7. `P6` 游戏内模型能力接入页  
8. `P7` 更多引擎兼容页面壳

### 第三阶段：平台化方向预留
9. `P8` 协作平台可视化  
10. `P9` 外部 AIGC 集成页  
11. `P10` Benchmark 与验收可视化

---

## 前端阶段的交付要求

每个功能在前端阶段至少要交付：

- 页面入口
- 信息架构
- 核心交互路径
- loading / empty / error / success
- destructive action confirmation
- mock 数据
- 状态切换逻辑
- 必要的接口契约草案
