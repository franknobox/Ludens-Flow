# Ludens Flow V2 架构重构与落地复盘

**时间**: 2026-02-24 至 2026-02-25
**核心目标**: 基于全新 V2 架构设计，打通从策划(GDD) -> PM排期 -> 架构建设(ENG) -> 审核打分(REVIEW) 的流水线状态机。重构底层 `orchestrator` 和 `router` 机制，扫清死锁并让自动化测试链路全线通关。

---

## 🎯 一、V2 架构蓝图与业务拆解

在 V2 的重塑中，我们彻底贯彻了“模块去耦、职责单一”的指导思想：

*   **四大 Agent 天王**：策划(`DesignAgent`)、项目经理(`PMAgent`)、技术专家(`EngineeringAgent`) 和铁面判官(`ReviewAgent`)。它们只负责在提示下生成具体的产物（Artifacts）并定稿提交，而不再关心整个流水线走到哪里。
*   **Router (路由/传送带)**：化身为完全无状态的规则字典。只根据当前系统处于的 `Phase` 以及刚收到的 `Last Event`（最新动作信号弹）判断“下一站去哪”。
*   **Orchestrator (场控/调度器)**：真正拉动循环的主力引擎。负责和 Agent 通信、拦截与保存磁盘物料，并拿着 Agent 给出的信号弹去问 `Router` 要下一步路线，最后更新大盘监控器（`state`）。

**预期畅通的业务骨干线**：
1. `GDD_DISCUSS` -> `GDD_COMMIT` (提交 GDD)
2. `PM_DISCUSS` -> `PM_COMMIT` (提交项目排期)
3. `ENG_DISCUSS` -> `ENG_COMMIT` (提交实现树架构)
4. `REVIEW` (进行打卡并生成审判录) -> `POST_REVIEW_DECISION`
5. 裁判决策：打回原点重新讨论 OR 绿灯放行进入下一代工序 `DEV_COACHING`。

## 🔨 二、重构阵痛与引擎级隐患破解

在上述架构打完地基试图连通时，我们迎来了最猛烈的系统性 Bug 扑击，其中有两个核心卡点。

### 1. `state.py` 优化与内聚
原有的 `WORKSPACE_DIR` 和路径处理逻辑散落、混乱且容易被操作系统分隔符坑害。我们将相关常量并拢至 `state.py` 和 `config.py`。
**成果**：简化了入口。测试或引擎在初始化的时候只需要调用 `st.init_workspace()` 并在完毕后 `st.load_state()` 就能安全获取互不污染的独立环境实例。

### 2. 致命的 "ENG_COMMIT 死锁"
当流程艰难地跋涉到 `ENG_COMMIT` 阶段，Agent 已经顺从地交出了代码大纲并高喊了那句 `ENG_COMMITTED`（已定稿），结果 Router 却表示“没收到”，流程陷入死循环。

**破案**：在追踪了成百上千条 Debug 日志后，我们定位到了 `orchestrator.py` 在接发信号时的低级却致命的漏传：
在提取到 Agent 返回的事件（event）后，只将其写在了一个局部变量 `last_event` 里，却**忘记将其挂载、写入大盘 `state.last_event` 中**。
**修复**：添加了那句至关紧要的 `state.last_event = last_event`，一举打通了这块淤堵在流转主动脉里的血栓。

## ✅ 三、测试收网与双线闭环

当底盘被敲打坚实，我们就遇到了最后一个难缠的对象——最严苛的、布满了各种拦截检查哨卡的验收测试套件 `test_step4_acceptance.py` 反复报红。

### 1. 拔除乱添的“测试拐杖”
我们发现，它前期报错多是帮我们逮出了真 Bug；但后期反复在终点站翻车，却是因为我们在排错期间，**生硬地加进去了好几个“强行空走一步” (`execute_step("")`)** 以图能看看究竟卡在哪。
而当底层机制被修成纯净无垢的光滑面后，这些用来催促死锁的“空推”让状态机跑得过快，一下越过了预设在 `REVIEW` 的检查哨进入了下一关，从而引起了代码断言崩落。我们大刀阔斧地将这些多余的空载指令**悉数砍光**。

### 2. Happy Path 的诞生与胜利
为了证明底层主心骨不再依赖修修补补，我们新建了一份名为 **`test_happy_path.py`**（纯净绿灯路线）的双盲文件。
在这个文件里，毫无断层、不用强推，纯天然地顺承调用，验证了这套系统目前可以在一个微秒级指令下，一路毫发无损地自动落盘四个关卡的全部物料，通畅到底。
随后，我们带着这份正确的时序和自信，回身几行代码就将原先的“魔鬼测试” `test_step4_acceptance.py` 满分开通！
