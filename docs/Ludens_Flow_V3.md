# Ludens-Flow V3 设计案（框架稿）

## 1. 文档定位

这份文档面向 V3 阶段的整体设计框架与方向展望。

它不替代当前的 [ROADMAP.md](/e:/GitDesktop/Ludens-Flow/docs/ROADMAP.md)，而是作为更上层的一版版本设计说明：
- `ROADMAP.md` 更偏当前阶段的持续推进项
- 本文档更偏 V3 的系统形态、能力框架和版本愿景

## 2. V3 愿景

Ludens-Flow V3 的目标，不只是把现有多 Agent 工作流继续扩展，而是把它进一步推进成一个更稳定、可管理、可演进的“项目工作台系统”。

在 V3 的设想中，Ludens-Flow 不再只是一个围绕单次流程运转的原型，而是一个具备明确层次、项目上下文和持续演进能力的协作底座。

V3 期望形成以下几类能力：
- 更稳定的工作流主链与状态流转
- 更明确的 Agent 职责边界与配置体系
- 更完整的多项目工作台与项目级上下文管理
- 更自然的前后端协作与多模态交互体验
- 面向知识增强、工程接入和评测体系的扩展基础

## 3. 设计框架

### 3.1 Workflow Core

V3 仍以 Design / PM / Engineering / Review 的主链为核心，但重点不再只是“能流转”，而是：
- 状态切换更清晰
- 异常恢复更稳定
- 工件读写与冻结边界更一致
- 跨阶段上下文传递更可控

这一层定义的是 V3 最基础的执行骨架。

### 3.2 Agent Configuration Layer

V3 希望把 Agent 的行为定义进一步结构化，而不是长期依赖散落在代码中的 prompt 和角色约定。

这一层主要承载：
- Agent 人设 / 系统提示配置
- 用户画像与用户偏好配置
- Prompt 注入机制
- 结构化输出约束（Schemas）

目标是让 Agent 行为具备更稳定的配置边界和复用能力。

### 3.3 Project Workspace Layer

V3 的一个重要方向，是把现有多项目工作台从“基础可用”推进到“完整可管理”。

这一层主要关注：
- 项目创建、切换、归档与索引
- 项目元数据体系
- 项目级状态、工件、日志、画像、记忆隔离
- 项目级 reset / migrate / lifecycle 管理

这一层决定 Ludens-Flow 是否真正具备“工作台系统”的形态。

### 3.4 Frontend Workbench Layer

V3 前端不只是展示窗口，而应成为多 Agent 协作的主要操作界面。

这一层主要包括：
- 项目视图与工件视图
- Agent 对话与只读历史切换
- 阶段状态、定稿状态、冻结状态展示
- 多模态输入反馈
- 更明确的确认、提示和防误触交互

前端目标不是堆叠界面，而是让项目状态、工件关系和操作路径更清晰。

### 3.5 Extension Layer

V3 需要预留扩展层，为后续能力演进提供统一入口。

候选方向包括：
- RAG / 知识检索增强
- Skill-based Agent Architecture
- Unity 工程文件接入
- Benchmark 与评测体系

这一层不要求在 V3 一次完成，但需要在设计上预留稳定接口。

## 4. 设计重点

如果把 V3 看成一版系统设计，而不是任务清单，那么它的重点可以概括为：

1. 从“单流程系统”走向“项目工作台系统”
2. 从“Prompt 驱动实现”走向“配置化、结构化实现”
3. 从“前后端勉强协作”走向“前后端围绕项目上下文稳定协作”
4. 从“局部功能堆叠”走向“面向扩展的整体架构”

## 5. 预期形态

V3 完成后，Ludens-Flow 期望具备这样一种形态：

- 它可以同时承载多个项目，而不是围绕单一运行态工作
- 它的 Agent 行为与输出约束更清晰，便于维护和演进
- 它的前端工作台可以真实支撑项目推进、工件查看和状态判断
- 它可以自然连接后续的知识增强、工程接入和评测体系

从更长的视角看，V3 不只是一次功能迭代，更像是把 Ludens-Flow 从“可运行原型”推进到“可持续演进的平台骨架”。

## 6. 与当前 Roadmap 的关系

当前 [ROADMAP.md](/e:/GitDesktop/Ludens-Flow/docs/ROADMAP.md) 中的大部分事项，都可以视为 V3 的组成部分。

对应关系大致如下：
- 稳定优化 -> Workflow Core
- 人设 / 用户画像 / Prompt / Schemas -> Agent Configuration Layer
- 多项目工作台化 -> Project Workspace Layer
- 前端配合性优化 -> Frontend Workbench Layer
- RAG / Skills / Unity / Benchmark -> Extension Layer

因此，这份 V3 文档更像是“版本设计框架”，而 `ROADMAP` 仍然是“推进路线图”。
