# Ludens-Flow 后续开发计划 (Roadmap)

## 1. 稳定优化
- **核心目标**：持续提升系统稳定性、状态流转一致性与整体可维护性，优先解决影响体验和可扩展性的基础问题。
- **具体方向**：
  - 梳理并收敛核心工作流中的状态切换、异常处理与日志行为，减少状态漂移和边界条件问题。
  - 优化多模态能力接入后的输入解析、错误兜底、数据流转与前后端协作体验。
  - 增加文件输入能力，优先支持纯文本、代码文件和常见文档输入，并完善解析、大小限制、编码处理与安全兜底。
  - 优化定稿生成的判断逻辑；优先采用显式确认，后续再引入语义识别作为辅助判断。
  - 继续清理冗余代码与历史遗留实现，降低后续功能扩展成本。
- **进度状态**：进行中

## 2. 独立系统人设与用户人设文件
- **核心目标**：将系统角色设定与用户画像配置解耦，形成可独立维护、可复用的配置体系。
- **具体方向**：
  - 提示词注入实现
  - 为 Design / PM / Engineering / Review 等 Agent 提供独立的人设或系统提示配置文件。
  - 引入用户人设文件，描述用户的技术背景、项目偏好、表达方式与协作期望。
  - 让系统根据不同用户画像动态调整对话风格、解释深度和输出重点。
- **进度状态**：用户人设文件基本完成

## 3. 提示词 (Prompt) 优化与结构化输出 (Schemas)
- **核心目标**：提升提示词稳定性、输出质量与工具调用一致性，减少非结构化输出带来的脆弱性。
- **具体方向**：
  - 分阶段优化核心 Prompt，使不同 Agent 的职责边界、输出目标和约束条件更清晰。
  - 推动关键节点输出结构化，逐步引入统一的 Schema 约束。
  - 为后续工具调用、前端渲染和自动校验提供更稳定的数据接口。
- **进度状态**：待启动

## 4. 多项目工作台化 (Multi-Project Workspace)
- **核心目标**：支持多个项目并行管理，让每个项目拥有独立的状态、工件、记忆与历史记录。
- **具体方向**：
  - 将当前单一 `workspace/` 结构演进为支持多项目隔离的工作台模型。
  - 支持项目创建、切换、归档与索引管理，避免每次重置都覆盖同一套运行数据。
  - 为前端和 CLI 提供统一的项目上下文切换能力。
- **进度状态**：待启动

## 5. 前端的配合性优化
- **核心目标**：让前端更好承接多 Agent 工作流，提高可视化协作、状态感知与操作效率。
- **具体方向**：
  - 优化前端与后端状态同步机制，降低界面状态与实际工作流状态不一致的问题。
  - 继续改善 Agent 视图、文件视图、阶段提示、只读控制和输入反馈体验。
  - 支持文件上传与解析结果展示，优先覆盖纯文本、代码和文档类输入场景。
  - 增加前端定稿确认交互，例如确认按钮、状态提示和二次确认流程，降低误触发与误提交风险。
  - 为多模态输入、结构化输出展示与后续多项目工作台能力预留前端接口。
- **进度状态**：进行中

## 远期规划 (Long-term Vision)

> 以下为探索性方向，不设硬性排期，将根据近期计划推进情况逐步展开。

## 6. RAG 知识检索增强 (Retrieval-Augmented Generation)
- **核心目标**：让 Agent 在回答与建议时具备可检索、可引用的外部知识增强能力。
- **具体方向**：
  - 支持对历史项目工件、用户画像和阶段产物做向量化索引与检索。
  - 引入 Unity 文档、开发规范或技术资料库作为辅助知识源。
  - 为后续更高质量的 Coaching、Review 和方案比较提供依据。

## 7. Agent 能力 Skills 化 (Skill-based Agent Architecture)
- **核心目标**：将 Agent 能力从大而全的 Prompt 中拆分出来，形成可组合、可扩展的能力模块。
- **具体方向**：
  - 设计统一的 Skill 接口与装配方式，降低能力扩展时对主流程的侵入。
  - 支持不同 Agent 在不同阶段按需加载 Skill，而不是依赖固定的一整套 Prompt。
  - 提升能力复用率，方便跨 Agent 共享最佳实践。

## 8. Unity 工程文件接入 (Unity Project Integration)
- **核心目标**：逐步从“对话式规划”走向“结合实际工程文件的开发辅助”。
- **具体方向**：
  - 支持读取 Unity 项目目录结构与代码文件，增强上下文感知。
  - 探索对 `.cs`、`.prefab`、`.unity` 等文件的分析与辅助能力。
  - 为后续更深入的代码生成、检查与项目协作能力打基础。
## 9. Benchmark 与评测体系构建 (Benchmark & Evaluation)
- **核心目标**：建立适配 Ludens-Flow 的评测体系，用可重复、可比较的方式衡量工作流质量、稳定性与真实任务完成度。
- **具体方向**：
  - 构建项目自定义 workflow benchmark，覆盖需求澄清、定稿确认、工件生成、Review 回流、多模态输入与异常场景处理。
  - 参考真实任务型 benchmark 与工具调用 benchmark，评估多 Agent 协作、结构化输出和工具使用质量。
  - 沉淀统一的评估指标，例如阶段推进成功率、误跳转率、工件完整性、响应时延与演示稳定性。
